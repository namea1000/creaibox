* 스마트 페이지 네이션
스마트 페이지네이션(Pagination) 탑재 (1페이지, 2페이지... 분할 제어):
원고가 수십, 수백 개로 늘어났을 때 밑으로 무한정 늘어지는 불편함을 완벽하게 방지하기 위해 한 페이지당 8개씩 원고를 정밀 분할 출력하도록 페이지네이션 알고리즘을 이식했습니다.
하단에 아름답게 세공된 처음 / 이전 / 1 / 2 / ... / 다음 / 끝 페이지네이션 컨트롤러를 달아두어 터치와 마우스 클릭으로 신속하게 오갈 수 있습니다.


*표(Table) 상시 고정 및 프리 렌더링 스켈레톤(Shimmer) 도입:
기존에는 로딩 중일 때 전체 표를 없애고 스피너를 보여주어 화면 깜빡임과 레이아웃 붕괴가 발생했습니다.
이제는 표의 외곽 구조와 헤더가 언제나 화면에 즉시 렌더링되어 고정됩니다.
데이터를 가져오는 동안에는 실제 원고와 똑같은 형태의 네온 실버 스켈레톤 행(Shimmer Skeleton Row 5개)이 부드럽게 반짝이며 렌더링되다가, 로드가 완료되면 글 제목과 내용들이 자연스럽게 채워집니다.


* 상태 필터링 기어
리스트 상단에 전체 원고, 임시 저장, 저장 완료, 발행 완료 탭을 정밀 연동하여, 각각 몇 개의 글이 저장되어 있는지 숫자로 실시간 현황이 집계되며 탭 선택 시 정확히 분류 필터링됩니다.


* 리얼 트랜잭션 액션 버튼:
발행(Publish): 클릭 즉시 Supabase 상의 원고 상태를 published로 업데이트하고 리스트 상태를 동기화합니다.
삭제(Delete): 안전 경고 팝업을 거쳐 Supabase DB에서 해당 행을 영구 소각한 후 리스트에서 실시간으로 지워냅니다.


* '초강력 HTML 테이블 구분선' 스와퍼 탑재!
이 문제를 해결하기 위해, 화면 프리뷰에서는 유려한 반응형 디자인을 유지하되, COPY 버튼을 누르는 0.01초의 순간에 구분선 코드를 네이버 에디터가 가장 정밀하게 인식하는 '정밀 HTML 3열 레이아웃 테이블(Table)' 구조로 실시간 치환하는 특수 치환 엔진을 장착했습니다.

초강력 인라인 DOM 파서 구축: 복사 동작을 시작하는 즉시 렌더링된 마크다운 엘리먼트를 스캔하여 네이버 스마트에디터가 인식하는 Malgun Gothic 폰트, 형광펜 구문, 마이크로 여백 규격, H1/H2 기둥 바 디자인 CSS를 태그에 수동으로 밀어 넣은 뒤 Rich HTML 규격으로 전달합니다.


* '초고화질 Rich Text' 복사
사장님! 보여지는 건 화면에서 정말 실제 블로그처럼 예쁘게 나오는데, 네이버 블로그 에디터(SmartEditor ONE)로 옮겨 붙여넣었을 때 ## 1. 교통비 걱정 끝!...처럼 마크다운 기호가 그대로 유출되는 현상이 왜 발생하는지 정확히 짚어내고 100% 해결해 드리겠습니다.
💡 왜 마크다운 기호(##, *, >)가 그대로 복사되었을까요?
이전의 COPY 버튼은 본문 데이터의 "순수 텍스트(Plain Text)" 값을 긁어가도록 짜여 있었습니다. 하지만 네이버 블로그 에디터는 메모장처럼 단순한 마크다운 문법 해석을 자동으로 해주지 않기 때문에 기호가 그대로 노출되었던 것입니다.
🚀 완벽한 해결책: '초고화질 Rich Text' 복사 엔진 장착!
이 문제를 해결하기 위해, 네이버가 가장 좋아하는 "웹 HTML 스타일"을 그대로 주입하여 클립보드에 박아주는 스마트 복사 엔진을 설계했습니다.
