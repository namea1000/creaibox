# 크리에이박스 X 네이티브 광고(Taboola) 수익 공유 모델(Revenue Sharing) 기획서

## 🌟 1. 비즈니스 개요 및 가치 제안

본 기획은 크리에이박스(CreAibox) 플랫폼 내 수백, 수천 명의 일반 사용자 블로그(`brand_id.creaibox.com` 및 연동 개인 도메인)에서 발생하는 대량의 트래픽을 네이티브 추천 광고 네트워크(Taboola, Outbrain 등)와 제휴하여 통합 수익화하고, 발생한 수익을 플랫폼과 사용자가 **5:5로 쉐어**하는 상생형 수익 구조(Revenue Share Model) 구축 방안을 정의합니다.

### 1.1. 기존 구글 애드센스 모델의 한계

* **높은 승인 허들**: 개별 블로그 단위로 구글 애드센스의 개별 심사(애드고시)를 통과해야 하므로 일반 초보 블로거의 90% 이상이 중도 포기합니다.
* **서브도메인 제한 정책**: 구글 정책상 `*.creaibox.com` 과 같은 서브도메인 구조는 독립된 심사가 까다롭고 거절 비율이 높습니다.

### 1.2. 타불라(Taboola) 제휴 모델의 압도적 강점

* **가입 즉시 무심사 수익화 (Zero-Barrier)**:
  * 크리에이박스 본사 도메인(`creaibox.com`)이 타불라와 1회 대표 파트너십을 맺으면, 플랫폼 하위의 모든 서브도메인 블로그에는 **아무런 개별 승인 절차 없이 광고 위젯이 가입 즉시 즉각 노출**됩니다.
* **사용자 강력한 락인(Lock-in) 효과**:
  * "글만 쓰면 즉시 돈이 벌린다"는 강력한 금전적 보상 기제를 제공하여 사용자의 글쓰기 활동 빈도를 비약적으로 높입니다.
* **AI 포스팅과의 무한 시너지**:
  * 크리에이박스의 올인원 AI 글쓰기 및 이미지 생성 툴로 양질의 기사를 빠르게 생성해 배포하면 ➡️ 검색 노출 트래픽이 발생하고 ➡️ 그것이 고스란히 광고 클릭 수익으로 자동 환원되는 **완벽한 선순환 파이프라인**을 제공합니다.

## 🛠️ 2. 시스템 아키텍처 및 구현 설계

전체 플랫폼의 광고 집행 및 정산 자동화를 위한 3대 핵심 아키텍처 파이프라인 설계안입니다.

```
[사용자 블로그 뷰어] ──(brand_id 태그 전송)──> [Taboola 광고 서버]
                                                     │ (광고 노출/클릭 집계)
                                                     ▼
[사용자 정산 대시보드] <──(일일 수익 적립)── [크리에이박스 백엔드 스케줄러]
                                            ▲
                                            │ (Sub-channel API 조회)
                                     [Taboola 파트너 API]
```

### 2.1. 광고 위젯 동적 추적 스크립트 삽입

사용자 블로그 포스트 하단 추천 기사 영역에 타불라 위젯 코드를 삽입하되, 어떤 사용자 블로그에서 수익이 났는지 구분하기 위해 `data-taboola-subchannel` 파라미터에 사용자 `brand_id` 혹은 `user_id`를 동적으로 바인딩합니다.

* **예시 스크립트 템플릿**:
  ```html
  <div id="taboola-below-article-thumbnails"></div>
  <script type="text/javascript">
    window._taboola = window._taboola || [];
    _taboola.push({
      mode: 'thumbnails-a',
      container: 'taboola-below-article-thumbnails',
      placement: 'Below Article Thumbnails',
      target_type: 'mix',
      // 🌟 사용자 식별을 위한 동적 서브채널 태깅
      subchannel: '{brand_id_parameter}' 
    });
  </script>
  ```

### 2.2. 정산 데이터 동기화 (Data Sync Scheduler)

매일 새벽 1회, 크리에이박스 백엔드 스케줄러(Cron Job)가 타불라 파트너 API를 호출하여 서브채널(`subchannel = brand_id`)별 전날의 예상 광고 수익(Revenue) 데이터를 긁어와 자사 DB 테이블에 누적합니다.

* **타불라 API 호출 URL 예시**:

  ```
  GET https://api.taboola.com/1.0/creaibox-publisher/reports/revenue-summary?subchannel=all&date=yesterday
  ```
* **신설 대상 데이터베이스 테이블 스키마 (`user_ad_revenues`)**:

  ```sql
  CREATE TABLE user_ad_revenues (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
      brand_id VARCHAR(100) NOT NULL,
      log_date DATE NOT NULL,
      raw_revenue NUMERIC(10, 2) NOT NULL,       -- 타불라 집계 전체 수익 ($ USD)
      user_share NUMERIC(10, 2) NOT NULL,        -- 사용자 몫 50% ($ USD)
      exchange_rate NUMERIC(8, 2) NOT NULL,      -- 당시 원/달러 환율
      converted_krw INTEGER NOT NULL,            -- 최종 사용자 적립 원화 (KRW)
      created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
  );
  CREATE UNIQUE INDEX idx_user_revenue_date ON user_ad_revenues (user_id, log_date);
  ```

## 📈 3. 사용자 정산 대시보드 및 환전 서비스 기획

사용자가 느끼는 보상의 가치를 시각적으로 극대화하기 위해 마이페이지 내에 프리미엄 디자인의 **[광고 수익 정산 센터]**를 제공합니다.

### 3.1. 화면 구성 요소 (UI Layout)

* **수익 현황 스코어보드**:
  * `누적 총 적립 수익 (KRW)` / `출금 가능한 잔액 (KRW)` / `어제 하루 수익 (KRW)`
* **출금 신청 폼**:
  * 잔액이 30,000원 이상일 때 활성화되며, 예금주명, 은행명, 계좌번호를 입력하고 **[출금 신청하기]** 버튼 클릭 시 관리자 콘솔로 즉시 정산 신청 접수.
* **적립 기록 테이블**:
  * 날짜별 노출수(PV), 예상 클릭수, 일일 정산금 리스트 그리드 출력.

### 3.2. 세무 및 원천징수 처리 표준 프로토콜

현금 정산(출금) 발생 시 대한민국 세법에 따라 원천징수 및 세무 신고 의무가 발생하며, 사용자의 사업자 유무에 따라 프로세스를 이원화하여 관리합니다.

#### 1. 개인 (비사업자 / 프리랜서) - 3.3% 원천징수 대상
* **세법 정의**: 인적용역 제공에 따른 사업소득(3.3%)으로 세무 신고를 대행 처리합니다.
* **정산 금액 계산 방식**: 
  - `실제 지급액 = 신청 정산액 - 원천징수세 3.3% (국세 3% + 지방세 0.3%)`
  - 예: 사용자가 100,000원 출금 신청 시 ➡️ 3,300원을 원천징수 공제하고 계좌에는 **96,700원**만 송금.
* **수집 및 신고**: 
  - 출금 신청 화면에서 최초 1회 **주민등록번호(실명인증 필수)** 및 주소를 필수 수집합니다.
  - 매월 10일 원천징수 신고서 및 지급명세서를 국세청 홈택스에 전자 제출하여 신고를 대행합니다.

#### 2. 개인사업자 및 법인사업자 - 세금계산서 발행 대상
* **세법 정의**: 부가가치세법상 사업용 재화/용역 매출이므로 세금계산서를 본사(크리에이박스)에 발행해야 합니다.
* **정산 금액 계산 방식**:
  - `실제 지급액 = 신청 정산액 + 부가세 10%`
  - 예: 사용자가 100,000원 출금 신청 시 ➡️ 본사(크리에이박스 사업자)를 공급받는자로 지정하여 110,000원(부가세 10,000원 포함)의 전자세금계산서를 발행. ➡️ 확인 즉시 **110,000원**을 계좌로 송금. (본사가 납부한 10%의 매입 부가세는 매입세액 공제로 돌려받으므로 본사 손익에 영향 없음)
* **수집 및 신고**: 
  - 최초 1회 **사업자등록증 사본** 및 **통장 사본**을 수집하여 검증합니다.

크리에이박스는 글로벌 1위인 타불라(Taboola) 외에도, 국내외 시장 점유율 및 광고 효율성을 다각도로 검토하여 최상의 파트너사를 확보하기 위한 다자간 경쟁 협상 전략을 추진합니다.

* **공식 웹사이트**: [https://www.outbrain.com](https://www.outbrain.com)
* **광고 신뢰도**: 사기성 광고나 불쾌감을 주는 낚시성 광고의 송출 제한이 엄격하여, 사용자 블로그의 시각적 품질과 신뢰도를 고급스럽게 보존할 때 우수합니다.

* **공식 웹사이트**: [https://www.mgid.com](https://www.mgid.com)
* **진입 장벽 최소화**: 제휴 신청 시 초기 요구하는 고정 트래픽 허들이 상대적으로 유연하여 플랫폼 초기 런칭 제휴에 유리합니다.

### 5.2. 비즈니스 협상 및 파트너 결정 전략

* **다자 구도 경쟁 비딩(Bidding)**:
  - 크리에이박스 플랫폼 내 통합 활성 트래픽이 임계치에 도달하는 시점에 **데이블(Dable)**과 **타불라(Taboola)** 파트너십 담당 부서에 동시 제휴 제안서를 송출합니다.
  - 양사에게 크리에이박스 플랫폼의 AI 기반 확장 규모와 서브채널 연동 기술을 증명하고 **"수익 쉐어 비율 상향 (예: 본사 60% vs 광고사 40%)"** 및 **"최소 보장 RPM 단가"** 비딩을 유도하여 가장 유리한 계약 조건을 쟁취합니다.
