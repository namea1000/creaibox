# CreAibox 배경 무인 자동화 및 아키텍처 가이드
(GitHub Actions & Supabase pg_cron 기반)

본 문서는 크리에이박스(CreAibox) 플랫폼의 **Vercel 자원 소모(CPU/실행시간) 0% 달성**과 **100% 무인 자동 수집 및 예약 작업**을 구현하기 위한 백그라운드 아키텍처 설계 명세서입니다.

---

## 1. 개요 및 배경 (Vercel Cron의 한계)

### 1.1 기존 Vercel Serverless Cron의 구조적 한계
- **실행 시간 제한 (Timeout 에러)**: Vercel 무료(Hobby) 서버리스 환경은 1회 함수 실행 시간이 10~15초로 매우 엄격하게 제약됩니다. 8대 국가 × 8개 카테고리(총 64회 이상 API 호출 및 쇼츠 리다이렉트 검증) 수집 시 타임아웃으로 인해 강제 종료되거나 수집이 실패했습니다.
- **서버 자원(GB-Hours) 소모 문제**: Vercel Cron이 돌 때마다 서버리스 인스턴스가 생성되어 CPU 및 메모리 사용량이 차감되며, 사용량이 누적될 경우 요금 청구 및 서비스 쿼터 차단 위험이 존재합니다.
- **해결 과제**: Vercel의 서버리스 자원을 1%도 소모하지 않으면서, 타임아웃 제약 없이 매일/매시간 완전 무인으로 동작하는 외부 백그라운드 스케줄링 구조로의 전환.

## 2. 무인 자동화 핵심 아키텍처 (Key Technologies)

### 2.1 GitHub Actions (권장 1순위 / 100% 무료 워크플로우 실행기)
- **개념**: GitHub 저장소 내부에 워크플로우 정의 파일(`.github/workflows/*.yml`)을 두고, GitHub이 자체 가상 서버(Runner)를 무료로 제공하여 스케줄(Cron)에 따라 스크립트를 자동 실행하는 방식입니다.
- **주요 특징 및 혜택**:
  - **Vercel 자원 소모 0%**: 모든 스크레이핑 및 API 연동 작업이 GitHub 서버에서 작동하여 Vercel CPU/Timeout 제약이 완전히 소멸합니다.
  - **월 2,000분 무료 (Private Repo 기준)**: 매일 2~3분 수집 실행 기준, 월 90분 내외만 사용되므로 평생 100% 무료로 운용 가능합니다. (Public 저장소는 무제한 무료)
  - **타임아웃 무제한**: 작업당 최대 6시간까지 연속 실행 가능하여 대용량 수집/분석/AI 처리 작업에 최적화되어 있습니다.
  - **실패 시 실시간 알림**: 수집 장애 시 개발자/관리자 이메일로 실패 로그 및 알림을 즉시 발송합니다.

### 2.2 Supabase `pg_cron` + `pg_net` (DB 내장 스케줄러)
- **개념**: Supabase PostgreSQL 데이터베이스 내부 확장 기능인 `pg_cron`(데이터베이스 스케줄러)과 `pg_net`(비동기 HTTP 요청)을 사용하여, DB 내부에서 지정된 시간에 쿼리를 실행하거나 외부 API를 호출하는 방식입니다.
- **주요 특징 및 혜택**:
  - 외부 서버나 Vercel 연결 없이 데이터베이스 수준에서 독립 실행됩니다.
  - 데이터 정경화, 기간 만료 처리, DB 내부 상태 변경 작업에 매우 유용합니다.

## 3. 백그라운드 자동화로 확장 가능한 무궁무진한 기능들

GitHub Actions와 Supabase 백그라운드 시스템을 도입하면 다음과 같은 고도화된 무인 자동화 기능들을 손쉽게 확장 구축할 수 있습니다.

| 순번 | 기능 명칭 | 설명 및 활용 방안 | 실행 주체 |
| :---: | :--- | :--- | :---: |
| **1** | **유튜브 급상승 트렌드 일괄 수집** | 매일 새벽 지정 시각에 8대 국가/8개 카테고리 급상승 비디오 64+ 개를 자동 수집하여 Supabase DB 및 구글 시트에 적재 | GitHub Actions |
| **2** | **AI 자동 블로그 예약 발행 (Auto Publisher)** | 인기 트렌드 키워드를 기반으로 매일 아침 AI 블로그 포스팅 초안을 자동 생성하고 예약된 시각에 블로그로 자동 발행 | GitHub Actions |
| **3** | **주간/일간 회원 맞춤 트렌드 리포트 메일링** | 구독 회원들에게 매주 월요일 아침 최신 인기 트렌드 리포트 및 바이럴 인사이트 요약본을 이메일(Resend 연동)로 자동 발송 | GitHub Actions |
| **4** | **검색엔진 자동 색인 신보 (Auto Indexing Ping)** | 새 블로그 글이 등록되면 Google Indexing API / Naver SearchAdvisor / Bing에 즉시 노출 수집 요청 핑(Ping) 전송 | Supabase / GitHub Actions |
| **5** | **회원 유료/VIP 혜택 만료 주기 자동 관리** | 만료 3일 전 회원에게 만료 안내 메일 발송, 만료 당일 회원 등급 자동 조정 및 사용 미완료 임시 데이터 자동 정돈 | Supabase `pg_cron` |
| **6** | **데이터베이스 백업 & 구글 드라이브 스냅샷** | 매주 일요일 밤 주요 테이블(유저, 원고, 이미지 생성 아카이브)을 엑셀/CSV로 내보내어 보안 안전 백업 저장소에 자동 저장 | GitHub Actions |

## 4. 로드맵 및 적용 가이드 (Implementation Roadmap)

### Phase 1. 유튜브 급상승 자동 수집기 이관 (GitHub Actions)
1. `.github/workflows/sync-trending.yml` 파일 생성.
2. 매일 KST 00:00 (UTC 15:00) 스케줄로 Node.js 수집 스크립트 실행.
3. Supabase Service Role Key를 GitHub Secrets에 등록하여 직접 DB에 결과 저장.

### Phase 2. 예약 발행 및 회원 주기 관리 고도화
1. 블로그 글 예약 발행 엔진 구현.
2. 회원 등급 및 만료 처리 자동화 구축.
