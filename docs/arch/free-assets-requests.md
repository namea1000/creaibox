# Free Asset Requests Architecture

## 1. Purpose
무료 공유 에셋 라이브러리(`free-assets`)에서 일반 사용자들이 공용으로 필요한 이미지를 관리자에게 요청하고, 관리자는 이를 접수 및 제작하여 완료 시 피드백을 전달할 수 있는 커뮤니티형 통로를 제공한다.
이를 통해 사용자는 필요한 에셋을 수급하고, 플랫폼은 사용자들이 어떤 에셋군을 필요로 하는지 시장 수요를 실시간 파악하며, 활발하게 에셋이 재생산되는 생태계를 구축할 수 있다.

---

## 2. Main Features
* **제작 요청 창구**: "무료 에셋 나눔하기" 좌측 버튼 클릭 시, 미디어 대분류 및 요청 명세를 입력할 수 있는 다이얼로그 모달 오픈.
* **공용 에셋 가이드 제공**: 모달 내에 개인 목적의 매우 상세한 이미지는 불가능하며, 모두가 함께 쓸 수 있는 공용 이미지에 한해 제작을 지원함을 명시.
* **요청 현황 사이드바**: 메인 에셋 리스트 우측에 2단 레이아웃(Aside) 형태로 전체 요청 현황과 진행 상태(`대기중`/`제작완료`)를 실시간 출력.
* **관리자 전용 답변 인터페이스**: 관리자로 로그인한 경우, 리스트의 각 대기 카드 하단에 답변 코멘트를 입력하고 완료 처리할 수 있는 컨트롤 활성화.

---

## 4. Database Structure
신규 생성된 `free_asset_requests` 테이블을 기반으로 작동하며, `profiles` 및 `admin_whitelist` 테이블과 조인하여 현재 접속자의 권한을 체크한다.
* `free_asset_requests`에 사용자의 이메일, 닉네임, 미디어 타입, 설명, 상태, 관리자 답변 등이 매핑되어 적재됨.

---

## 5. API Structure
* **`GET /api/free-assets/requests`**: 최신 제작 요청 목록을 반환. (비로그인 허용)
* **`POST /api/free-assets/requests`**: 로그인 회원 자격으로 신규 제작 요청을 생성.
* **`POST /api/free-assets/requests/comment`**: 관리자 권한을 검증한 뒤 요청 건에 코멘트를 등록하고 상태를 `completed`로 즉시 업데이트.

---

## 6. Component Structure
* **`FreeAssetsPage`** (`page.tsx`): 전체 페이지 오케스트레이션 및 레이아웃 정의 (aside 우측 정렬).
* **`RequestModal`**: 이미지 제작 요청 제출용 UI 및 안내 가이드 노출.
* **`RequestSidebar`**: 요청 건 목록 렌더링 및 각 요청 카드의 뱃지 색상, 관리자 댓글 노출 제어.
* **`AdminCommentForm`**: 관리자 로그인 상태에서만 마운트되는 인라인 코멘트 전송 및 완료 변경 컴포넌트.

---

## 7. Future Expansion
* **알림 연동**: 관리자가 완료 코멘트를 등록하면, 요청했던 사용자에게 내부 알림(Notification System) 또는 가입 이메일로 자동 완료 알림을 전송.
* **공감/추천 기능**: 다른 사용자들이 대기 중인 요청 사항에 "좋아요/나도필요해요"를 눌러, 가장 수요가 높은 요청을 최상단으로 정렬하는 투표 시스템 도입 가능.
