# 커뮤니티 스튜디오 (Community Studio)

이 문서는 크리에이터 전용 소통 공간인 커뮤니티 스튜디오 모듈의 아키텍처 및 기능을 설명하는 상시 운영용 문서입니다.

---

## 1. 목적 (Purpose)
* 크리에이터들이 스튜디오 대주제별로 세분화된 10개의 채팅방에서 실시간으로 대화하고 정보와 작업물을 공유할 수 있는 인터랙티브 협업 커뮤니티 공간을 구축합니다.
* 사용자가 직접 입력한 질문이나 메시지에 다른 크리에이터들이 반응하는 인터랙티브 환경을 클라이언트 단에서 생동감 있게 연계 제공함으로써, 플랫폼 내 활발한 참여 문화를 지원합니다.

---

## 2. 주요 기능 (Main Features)
1. **10대 테마 채널 분기**: 전체 채팅로비, 글쓰기, 네이버 블로그, 뮤직, 이미지, 비디오, 유튜브, AI 트렌드, 협업, 수익화 등 각 창작 도메인별 소통 채널 제공.
2. **동적 파라미터 라우팅**: Next.js App Router의 `[section]` 세그먼트 매개변수를 참조해, 사용자가 클릭한 채널로 즉각 진입하고 메시지 컨텍스트를 스위칭합니다.
3. **실시간 온라인 크리에이터 그리드**: 채널 전문성에 맞는 크리에이터들의 가상 활동 상태(온라인, 부재중 등) 목록을 우측 패널에 노출.
4. **대화 내용 자동 동기화**: 채널별 사전 정의된 3-4개 실감형 대화 데이터팩을 최초 진입 시점에 렌더링.
5. **AI 동료 크리에이터 오토리플라이**: 사용자가 글을 전송하면 해당 분야 가상 크리에이터의 프로필과 뱃지를 빌려 1.5초 후 컨텍스트 맞춤 피드백 또는 격려 메시지 자동 출력.

---

## 3. UI 및 동적 라우팅 (Routing Structure)
* **메인 랜딩**: `/studio/community/page.tsx`
  * 커뮤니티 주요 현황 요약 보드 및 10개 실시간 테마 채팅방 입장 카드 리스트.
* **상세 채팅방**: `/studio/community/[section]/page.tsx`
  * 동적 경로를 읽어 `ChatRoom.tsx` 컴포넌트를 마운트하고, 왼쪽 채널 사이드바, 중앙 메시지 패널, 오른쪽 온라인 크리에이터 목록을 3단 분할 그리드로 결합.

---

## 4. 데이터베이스 및 API (Database & API Structure)
* 현 단계에서는 오프라인 상태 및 초기 진입 반응 속도를 극대화하고 서버 비용을 최적화하기 위해, 로컬 파일 `chat-data.ts` 및 React Client State를 통해 데이터를 제어합니다.
* 향후 Supabase Realtime 또는 WebSockets 모듈과의 연동을 염두에 두고, 메시지 구조를 DB 스키마(`id`, `sender`, `avatarColor`, `content`, `timestamp`, `isUser`, `badge`)와 호환되게 구성했습니다.

---

## 5. 컴포넌트 구조 (Component Structure)
* `src/app/studio/community/[section]/`
  * `page.tsx`: 라우터 세그먼트 핸들러.
  * `components/ChatRoom.tsx`: 채널 사이드바, 메시지 창, 인풋 바, 크리에이터 목록을 바인딩하는 통합 대시보드 컴포넌트.
  * `lib/types.ts`: `Message`, `Creator`, `Channel`의 타입 선언부.
  * `lib/chat-data.ts`: 10개 채널 및 크리에이터 목록, 기본 대화 내용, 자동 완성 오토리플라이 스크립트.

---

## 6. 향후 확장 계획 (Future Expansion)
* **Supabase Realtime 채팅 동기화**: 실시간 사용자 웹소켓 브로드캐스팅 및 DB 영속화 기능 확장.
* **실제 파일/이미지 업로드**: 크리에이터 제작 작업물(음악 파일, 이미지 등)을 Drag & Drop으로 업로드하고 즉시 공유하는 미디어 프리뷰 카드 추가.
