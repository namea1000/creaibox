# Sitemap.xml (사이트맵) VS /feed (RSS 피드) 개념 및 검색 노출 최적화(SEO) 가이드

본 문서는 웹사이트 및 블로그 운영 시 검색 엔진(구글, 네이버 등)에 글을 더 신속하고 완벽하게 색인(Index)시키기 위해 반드시 활용해야 하는 두 가지 핵심 수단인 **Sitemap.xml**과 **RSS 피드(/feed)**의 차이점을 정의하고 사용 가이드를 제공합니다.

---

## 📂 1. 한눈에 보는 요약 비교

| 구분 | 📑 Sitemap.xml (사이트맵) | 📦 /feed (RSS 피드) |
| :--- | :--- | :--- |
| **비유** | 책의 맨 앞/뒤에 있는 **"전체 목차"** | 매일 아침 문 앞으로 배달되는 **"신문 배달"** |
| **대상 범위** | 사이트 내 존재하는 **모든 페이지 주소** (과거~현재 전체) | 사이트에 등록된 **최근 10~20개의 최신 글** (요약 정보 포함) |
| **데이터 크기** | 사이트 규모에 따라 상대적으로 **크고 무거움** | 최신 정보만 담아 대단히 **작고 가벼움** |
| **갱신 주기** | 부정기적 / 대량 페이지 변경 시 업데이트 | 새 글 발행 시 실시간으로 즉시 갱신 |
| **주요 역할** | 사이트의 **전체 골격과 모든 경로**를 검색 로봇에 안내 | 신규 작성된 **새 글을 실시간으로 빠르게** 검색 엔진에 전송 |

---

## 📑 2. Sitemap.xml (사이트맵) 상세

### 2.1. 개념 및 목적
사이트맵은 검색 엔진 크롤링 로봇(Googlebot, Yeti 등)에게 웹사이트의 구조를 계층적으로 보여주는 **지도(Map)** 역할을 합니다. 사이트 내에 숨겨진 페이지나 링크 연결이 다소 불완전한 깊숙한 경로의 글까지 누락 없이 모두 수집하도록 설계된 XML 규격 파일입니다.

### 2.2. 검색 엔진에서의 작동 방식
* 검색 엔진 로봇은 사이트맵을 통해 사이트의 전체 주소 목록을 받아 간 후, 변경 사항(수정 날짜 `<lastmod>`, 우선순위 `<priority>`)을 바탕으로 크롤링 우선순위를 설정합니다.
* 사이트 규모가 커질수록 사이트맵은 주기적으로 긁어가기엔 용량이 비대해져서, 검색 로봇이 보통 몇 주 혹은 몇 달 주기로만 이 파일을 정밀 재조회합니다.

---

## 📦 3. /feed (RSS 피드) 상세

### 3.1. 개념 및 목적
RSS(Really Simple Syndication)는 웹사이트의 최신 업데이트 정보를 사용자 또는 기계가 쉽게 구독해 읽을 수 있도록 표준화된 XML 템플릿입니다. 크리에이박스에서는 `/feed` 경로를 호출하면 최신 글들의 제목, 작성자, 내용 요약, 발행일 정보가 구조화되어 즉각 출력됩니다.

### 3.2. 검색 엔진에서의 작동 방식
* 최신 글만 담겨 있어 용량이 수 KB 이내로 극히 가볍습니다.
* 구글과 네이버 검색 로봇은 가볍고 즉각적인 RSS 피드 주소를 **매우 빈번하게(수 분~수 시간 주기) 스캔**합니다.
* 따라서 새로운 글을 쓰자마자 검색 엔진에 즉시 반영(색인 생성)되게 하려면 사이트맵과 함께 RSS 피드를 반드시 제출해야 합니다.

## 🛠️ 4. 크리에이박스에서의 구현 및 검색엔진 등록 가이드

크리에이박스로 만들어진 브랜드 블로그는 수동으로 파일을 업로드할 필요 없이 **실시간 가상 라우팅을 통해 사이트맵과 RSS 피드를 자동으로 자동 컴파일하여 서빙**합니다.

* **사이트맵 주소**: `https://대표도메인/sitemap.xml`
* **RSS 피드 주소**: `https://대표도메인/feed`

### 4.1. 구글 서치콘솔 (Google Search Console) 등록법
1. 구글 서치콘솔 로그인 후 좌측 메뉴의 **[색인 생성] > [Sitemaps]**로 이동합니다.
2. '새 사이트맵 추가' 영역에 다음 두 가지 주소를 각각 한 번씩 입력하여 제출합니다:
   * `sitemap.xml` 입력 후 제출
   * `feed` 입력 후 제출
3. '제출된 사이트맵' 목록에 두 항목의 상태가 **[성공]**으로 녹색 표시되는지 확인합니다.

### 4.2. 네이버 서치어드바이저 (Naver Search Advisor) 등록법
1. 네이버 서치어드바이저 웹마스터 도구 로그인 후 본인 도메인을 선택합니다.
2. 좌측 메뉴의 **[요청] > [사이트맵 제출]**에서 `sitemap.xml`을 입력하고 확인을 누릅니다.
3. 좌측 메뉴의 **[요청] > [RSS 제출]**에서 도메인을 포함한 전체 주소인 `https://본인도메인/feed`를 정확히 입력하고 확인을 누릅니다.
