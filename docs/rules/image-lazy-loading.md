# 이미지 지연 로딩 (Lazy Loading) 최적화 기술 가이드

크리에이박스(`CreAIbox`) 미디어 라이브러리와 같이 수천 장 이상의 고화질 이미지를 다루는 서비스에서 **초기 렌더링 성능 최적화** 및 **서버 트래픽 요금 절감**을 실현하기 위한 핵심 기술인 `loading="lazy"` 속성에 대해 설명합니다.

---

## 1. 개요 및 작동 원리

`loading="lazy"`는 HTML5 및 모던 브라우저 표준 명세에 포함된 네이티브 **이미지 지연 로딩(Lazy Loading)** 기술입니다.

### 작동 메커니즘
1. **렌더링 대기**: 웹 페이지가 로드될 때 브라우저는 HTML DOM 트리 내의 모든 이미지 태그를 해석합니다.
2. **뷰포트 감지**: 이때 화면(Viewport)에 직접 노출되어 있지 않은(스크롤 아래쪽 깊은 곳에 있는) 이미지들에 대해서는 **네트워크 다운로드 요청을 즉시 날리지 않고 대기(Wait)** 시킵니다.
3. **스크롤 접근 시 로드**: 사용자가 마우스 스크롤을 내려 해당 이미지 위치가 뷰포트 기준 일정 거리(브라우저가 판단하는 임계값) 안으로 들어오면, 브라우저가 실시간으로 백그라운드 다운로드를 실행해 이미지를 화면에 부드럽게 노출시킵니다.

---

## 2. 왜 필수적인가?

### ① 서버 트래픽 비용 극대화 절감 (0원화 지향)
사용자가 미디어 라이브러리에 진입한 후 첫 페이지만 보고 나갈 경우, 아래쪽에 있는 수백 장의 고해상도 이미지 데이터(수백 MB ~ 수 GB)를 미리 다운로드할 필요가 없습니다. `loading="lazy"`는 오직 **사용자가 직접 본 이미지만 요금을 지불**하도록 보장합니다.

### ② 클라이언트 메모리 및 렌더링 성능 가속화
브라우저가 한 번에 수백 개의 네트워크 커넥션을 열어 대용량 데이터를 내려받는 병목 현상이 제거되므로, 첫 화면 진입 속도가 **0.1초 수준**으로 획기적으로 개선됩니다.

## 3. 실제 코드 적용 예시

### 일반 `<img>` 태그 사용 시 (현재 라이브러리 적용 방식)
```tsx
<img
  src={asset.thumbnailUrl || asset.url}
  alt={asset.title}
  loading="lazy" // 👈 네이티브 지연 로딩 활성화
  className="w-full h-auto object-cover rounded-lg"
/>
```

### Next.js `<Image>` 컴포넌트 사용 시
Next.js의 `<Image>` 컴포넌트는 기본적으로 지연 로딩(`loading="lazy"`)이 내장되어 있습니다. 단, 첫 화면에 곧바로 보여야 하는 최상단 배너나 히어로 이미지의 경우 오히려 로딩이 지연되면 UX가 저하되므로 `priority` 속성을 주어 지연 로딩을 비활성화하는 것이 권장됩니다.
```tsx
import Image from 'next/image';

// 일반적인 목록 이미지 (지연 로딩 적용)
<Image
  src={asset.thumbnailUrl}
  alt={asset.title}
  width={400}
  height={300}
/>

// 최상단 메인 배너 이미지 (즉시 로드)
<Image
  src="/hero-banner.png"
  alt="Main Banner"
  width={1200}
  height={400}
  priority // 👈 즉시 강제 로딩
/>
```

## 4. 모던 브라우저 호환성
* Chrome, Edge, Safari, Firefox 등 현재 상용화된 **모든 모던 브라우저에서 98% 이상 완벽하게 표준 지원**합니다.
* 지연 로딩을 지원하지 않는 극소수 구버전 브라우저의 경우, 오류를 내지 않고 일반 이미지 로딩(`loading="eager"`)으로 자연스럽게 폴백(Fallback)되어 안전하게 출력됩니다.
