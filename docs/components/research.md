# 자료 조사 컴포넌트 (Research Studio Components)

웹 리서치, 문서 스크래핑, 그리고 수집한 자료 기반으로 AI 어시스턴트와 문답하는 리서치 스튜디오 모듈의 구성 및 상세 로직입니다.

---

## 1. 리서치 프로젝트 관리 대시보드 (`projects/page.tsx`)
* **역할**: 크리에이터가 도메인별(예: 신제품 분석, 유튜브 채널 연구 등)로 생성한 리서치 세션을 카드 타일 형태로 탐색.
* **통계 연산**: 
  - 각 프로젝트의 ID를 식별값 삼아 `research_sources`(수집자료), `research_chats`(대화방), `research_generated_contents`(결과글), `research_images`(추출 이미지) 테이블에서 현재 유저가 쌓아 올린 레코드들의 카운터를 실시간 크로스 분석해 출력합니다.
* **삭제 룰**: 단일 프로젝트 삭제 단추 클릭 시 Cascade 제약 조건에 준해 프로젝트에 종속되었던 하위 리서치 문서 및 채팅 기록이 한꺼번에 청소됩니다.

---

## 2. 웹/문서 리서치 생성 및 추출기 (`create/page.tsx`)
* **역할**: 키워드 연구, 파일 분석, 외부 웹 사이트 자료조사를 수립하기 위한 리서치 마법사 폼.
* **파싱 중계 흐름**:
  - **웹 사이트 주소 입력**: `/api/research/extract-url` (서버 cheerio 스크래퍼)을 경유하여 텍스트 정제.
  - **PDF 보고서 첨부**: `/api/research/extract-pdf` (서버 pdf-parse)를 경유하여 파일 내의 글자 블록 디코딩 및 추출.
* **DB 기록**: 
  - 생성 완료 시 `research_projects` 테이블에 신규 행을 만들고, 추출된 텍스트 본문 원본을 `research_sources` 및 `research_extractions` 테이블에 적재해 AI 채팅에서 문맥으로 쓰일 수 있도록 영속화합니다.

---

## 3. 리서치 기반 대화 채널 (`chat/page.tsx`)
* **역할**: 수집된 연구 원천 데이터를 문맥(Context)으로 주입해 질문에 사실 기반으로 답변하는 챗봇 패널.
* **작동 구조**:
  1. 사용자가 질문을 던지면, 프로젝트 ID에 묶인 `research_sources`의 텍스트 조각들을 조회하여 가장 관련성 높은 문서를 추출합니다.
  2. 추출된 텍스트를 LLM(Gemini 등)의 프롬프트 전반부 시스템 메시지에 **[참조 자료]** 형식으로 병합해 그라운딩(Grounding)을 유도합니다.
  3. 이로써 AI가 허위 정보(환각, Hallucination)를 사실인 양 지어내어 답변하는 문제를 최소화합니다.
  4. 질문과 응답 내역은 `research_chats` 및 `research_chat_messages` 테이블에 차례로 아카이빙됩니다.
