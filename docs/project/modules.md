# 모듈 상세 설명 (Modules Documentation)

`CreAibox` 프로젝트는 역할과 기능에 따라 크게 **Studio Module (제작자용)**, **Admin Module (관리자용)**, **Public Pages (일반 사용자용)**, 그리고 **Data/API Layer**의 4가지 주요 모듈로 나뉩니다.

---

## 1. Studio Module (제작자용)
크리에이터가 실제로 콘텐츠를 기획하고 생산하는 최상위 핵심 모듈입니다.

```
src/app/studio/
├── layout.tsx                # 스튜디오 좌측 사이드바 및 레이아웃 정의
├── dashboard/                # 스튜디오 전체 대시보드
├── research/                 # 키워드/자료 리서치 (프로젝트, 채팅, 이미지, 라이브러리)
├── music/                    # AI 음악 제작 (기획, 가사/SUNO, 앨범, 라이브러리, 비주얼라이저)
├── video/                    # 고급 비디오 에디터 (타임라인 기반 편집 및 내보내기)
├── writing/                  # AI 포스팅 글쓰기 (네이버 블로그, 워드프레스, 크리에이박스)
└── content-planner/          # AI 콘텐츠 플래너 & 아이디어 허브
```

### 1-1. AI 음악 제작 (Music Studio)
- **곡 기획 (`planning`)**: 앨범 방향성 설계 및 장르/콘셉트 구성.
- **가사 및 SUNO (`lyrics`)**: AI를 활용해 곡의 가사 및 SUNO 프롬프트, 묘사글을 자동 생성.
- **오디오 비주얼라이저 (`visualizer`)**: 브라우저 내에서 음악에 맞춰 반응하는 오디오 스펙트럼 캔버스 가동 및 WASM 렌더링.
- **앨범 관리 (`albums`)**: 다수의 곡을 앨범 단위로 패키징.

### 1-2. 비디오 에디터 (Video Studio)
- **비디오 에디터 (`editor`)**: 타임라인 레이어 제어, 미디어 라이브러리, 자막 레이어, 오디오 믹서, 화면 트랜지션, 텍스트 효과 기능을 내장한 SPA 비디오 편집 프로그램.
- **FFmpeg 렌더러**: 클라이언트 브라우저 단에서 FFmpeg.wasm을 활용하여 WebM 클립을 MP4로 트랜스코딩 및 빌드.

### 1-3. AI 포스팅 글쓰기 (Writing Studio)
- **네이버 글쓰기 (`naver`)**: 네이버 검색 키워드 분석 매트릭스를 연동하여 스마트 블로그 원고 작성.
- **워드프레스 글쓰기 (`wp`)**: 워드프레스 REST API 설정을 기반으로 임베디드 이미지 삽입, 썸네일 생성 및 다이렉트 퍼블리싱 지원.
- **크리에이박스 글쓰기 (`CreAibox`)**: SEO 최적화 패널, 스키마 정의 마크업, 포스트 슬러그 빌더를 내장한 블로그 에디터.

### 1-4. AI 콘텐츠 플래너 (Content Planner & Idea Hub)
- **아이디어 허브 (`idea-hub`)**: 카테고리/상세분야 분류 및 추천 시리즈를 필터링하여 기획 소재용 메인 키워드 주제를 탐색하는 대화형 대시보드. SEO 최적화와 글로벌 확장을 위한 한/영 병기 표기, 연도 자동 제거(Evergreen) 및 다국어 검색 치환 레이어를 제공.
- **콘텐츠 기획실행 (`planning`)**: 선택한 키워드로 목표(조회수, 수익화 등), 플랫폼(블로그, 유튜브, 틱톡 등), 어조 및 상세 조건을 입력하여 AI 콘텐츠 기획서(캠페인 및 개별 아이템)를 동적 자동 생성.

## 2. Admin Module (관리자용)
시스템 상태 및 요금제, 보안 API 키 관리를 위한 어드민 페이지 모듈입니다.

* **통계 (`analytics`)**: Google Analytics 및 Stripe 실 결제 통계를 조회하고, 전체 유입 채널 분석 및 실시간 트래픽을 관측하는 모바일-친화 대시보드.
* **API 금고 (`apivault`)**: Gemini, OpenAI, Claude, Groq 및 Google Search, Naver API 키를 등록하고 암호화하여 공용 한도로 관리하는 패널.
* **사용자 관리 (`usermanagement`)**: 가입 계정 목록 조회 및 어드민 권한 조율.
* **SEO 및 시스템 설정 (`seo`, `system`, `settings`)**: 로컬 웹 환경의 기본 설정값, 메타 검색 노출 태그 제어.

---

## 3. Public Pages (일반 사용자용)
일반 비로그인/신규 방문자가 이용하는 페이지 세트입니다.

* **메인 랜딩 페이지 (`/`)**: 앱 서비스 핵심 기능 설명 카드 및 가격 페이지 연결.
* **요금제 안내 (`pricing`)**: 멤버십 및 이용 등급표 시각화.
* **인증 페이지 (`login`, `signup`)**: Supabase Auth 연동을 통한 이메일 회원가입 및 간편 로그인.
* **블로그 및 홍보 (`blog`, `about`)**: 플랫폼 관련 안내 원고 및 업데이트 뉴스룸.
* **고객 지원 (`help`, `terms`, `privacy`)**: 이용약관, 개인정보처리방침, FAQ 목록.

---

## 4. Data / API Layer
프론트엔드와 외부 솔루션, DB를 중계하는 내부 브리지 API 및 쿼리/스토어 라이브러리입니다.

* **상태 스토어 (`src/lib/stores/`)**:
  - `manuscripts.ts`: Zustand를 이용해 네이버 및 크리에이박스 원고 상태(목록, 수정 사항, 임시 저장) 관리.
  - `manuscript-ui.ts`: 탭 전환 및 에디터 패널 레이아웃 조작.
* **서버리스 API 엔드포인트 (`src/app/api/`)**:
  - `/api/ai/generate`: 서버리스 환경에서 공용 API 키 풀을 선회하며 에러 발생 시 자동 Failover 복구를 수행해 텍스트를 리턴.
  - `/api/research/extract-pdf`, `extract-url`: cheerio 및 pdf.js를 사용한 외부 문서 웹 파싱 엔진.
