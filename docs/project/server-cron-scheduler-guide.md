# 서버 자동 스케줄러(Cron) 기술 비교 및 크리에이박스 자동화 아키텍처 가이드

본 문서는 크리에이박스(creaibox.com)의 정기 데이터 수집, 자동화 콘텐츠 발행 및 관리 시스템 구축을 위한 서버 자동 스케줄러(Cron) 기술 대안의 장단점 비교와 향후 자동화 시나리오 로드맵을 정리한 아키텍처 가이드입니다.

---

## 1. 3대 서버 자동 스케줄러(Cron) 기술 대안 비교

### 1.1 Vercel Cron (Next.js 어플리케이션 레이어 스케줄링)
* **구현 아키텍처**:
  * Next.js 어플리케이션이 배포된 Vercel 환경 내에서 설정 파일(`vercel.json`)에 cron 표현식 스케줄을 명시합니다.
  * 지정된 시각에 Vercel 엣지 네트워크가 프로젝트 내부의 보안 백엔드 API 엔드포인트(예: `/api/cron/sync-trending`)를 자동으로 트리거(GET/POST)하여 구동시킵니다.
* **장점**:
  * Next.js 어플리케이션 코드베이스와 완전히 통합되어 있어 개발자 생산성이 가장 높습니다.
  * 기존 프로젝트 내 정의된 데이터베이스 어댑터(Supabase 클라이언트), 외부 유튜브 API 헬퍼, 구글 스프레드시트 연동 함수(`appendTrendingToSheet`) 등을 수정 없이 그대로 임포트하여 재사용할 수 있습니다.
* **단점**:
  * Vercel의 Serverless 함수 기본 최대 실행 시간(Plan에 따라 10초~60초)의 영향을 강하게 받습니다. 8대 카테고리의 랭킹 수집 및 쇼츠 2차 리다이렉트 HEAD 요청이 많을 경우 타임아웃을 유발할 수 있으므로, 비동기 병렬 처리(`Promise.all`)를 적용하거나 엔드포인트를 카테고리별로 분할 호출하는 예외 처리가 요구됩니다.

### 1.2 Supabase pg_net / pg_cron (데이터베이스 레벨 스케줄링)
* **구현 아키텍처**:
  * Supabase 데이터베이스 엔진 내부에 설치된 `pg_cron` 익스텐션과 `pg_net` 익스텐션을 활성화합니다.
  * DB 내부 스케줄러 테이블에 크론 시간과 타겟 HTTP 요청을 정의하여 데이터베이스 레벨에서 직접 주기적으로 크리에이박스 백엔드 API 엔드포인트를 찌르거나 내부 SQL 트리거 함수를 실행합니다.
* **장점**:
  * 외부 크론 인프라에 대한 의존성이 제로이며 Supabase 콘솔 내부에서 데이터베이스 뷰와 함께 한 번에 통합 관리할 수 있습니다.
  * 수집 작업 후 바로 다른 DB 테이블을 조작하거나 연계 트랜잭션을 수행할 때 지연 시간이 거의 없고 안정적입니다.
* **단점**:
  * SQL 스크립트 작성 양식에 의존하므로 Next.js(TypeScript) 비즈니스 코드를 직접 삽입하기 어려워 API 연동 구조가 2중화될 수 있습니다.
  * API 호출 시 보안용 토큰 검증 로직을 SQL의 `pg_net` 헤더 설정에 담아 관리해야 하므로 설정 관리가 다소 복잡합니다.

### 1.3 Supabase Edge Functions (Edge Serverless Worker 기반)
* **구현 아키텍처**:
  * Deno 런타임 기반의 Supabase Edge Functions로 수집 작업용 독립 함수를 작성한 뒤, Supabase 플랫폼의 정기 Trigger Schedules 기능을 셋업해 구동시킵니다.
* **장점**:
  * Edge 물리 인프라에서 가볍게 분산 동작하여 어플리케이션 부하를 주지 않고, 호출 및 처리 속도가 최상위 수준으로 신속합니다.
  * 함수 호출당 단가가 매우 낮아 비용적인 최적화에 탁월합니다.
* **단점**:
  * Node.js 환경이 아닌 Deno 환경이므로 타겟 NPM 패키지나 모듈들의 미세한 호환성 체크를 선행하여 대응해야 합니다.

## 2. 크리에이박스(creaibox.com)의 향후 자동화(Cron) 확장 로드맵

정기 스케줄러 인프라가 배포되면 유저 인터랙션 없이도 플랫폼 스스로 작동하는 **무인 콘텐츠 자동화 파이프라인**을 신설할 수 있습니다.

### 2.1 트렌드 기반 AI 자동 블로그 포스팅 발행 (Auto-Publishing)
* **시나리오**: 매일 아침 7시, 크론이 8대 카테고리의 1위 급상승 비디오의 키워드 및 대상을 크롤링합니다. 이후 Gemini AI 프롬프트에 실시간 트렌드 메타데이터를 공급하여 분석 및 평론 요약글 초안을 작성한 뒤, 크리에이박스 공식 블로그나 워드프레스에 API 동기화로 자동 발행을 수립합니다.
* **비즈니스 가치**: 100% 무인 작동하는 SEO 최적화 트래픽 확보 파이프라인이 매일 생성됩니다.

### 2.2 크리에이터 타겟 '일일 트렌드 분석 보고서' 이메일 발송
* **시나리오**: Supabase DB와 구글 스프레드시트에 영구 축적된 랭킹 타임라인 데이터를 백서버가 정산하여, "전일 대비 급격하게 떠오른 비디오 키워드 및 연관 해시태그 분석표"를 작성합니다. 이후 매일 오전 8시 메일 서비스를 통해 크리에이박스 유료 멤버십 가입자들에게 뉴스레터를 자동 배송합니다.
* **비즈니스 가치**: 크리에이터들의 핵심 니즈를 저격하는 프리미엄 유료 정구독(뉴스레터) 비즈니스 모델을 도입할 수 있습니다.

### 2.3 라이벌 채널 통계 모니터링 및 실시간 추적 (Competitor Tracking)
* **시나리오**: 크리에이터가 본인의 작업실 대시보드에 등록한 벤치마킹 대상 경쟁 채널들의 유튜브 API 데이터를 매일 밤 12시 스캔합니다. 누적되는 구독자 수 변동 추이와 주간 신규 동영상 업로드 수 등을 Supabase DB에 축적하고 대시보드 그래프로 제공합니다.
* **비즈니스 가치**: 단순 분석기에서 경쟁사 타겟의 전문 분석 도구(SaaS)로 고도화됩니다.

### 2.4 미디어 에셋 라이브러리 정기 최적화 및 유휴 캐시 정리
* **시나리오**: 매주 일요일 새벽 2시, 무료 미디어 에셋 라이브러리(`creaibox-free-assets`)에서 다운로드 수 및 조회수 통계를 정산하여 주간 에셋 인기 점수를 산출합니다. 또한, 장기 미사용 썸네일 캐시나 정리가 필요한 임시 데이터들을 정기적으로 삭제하여 스토리지 보관 비용을 절감합니다.
