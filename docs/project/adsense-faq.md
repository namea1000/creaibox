# 구글 애드센스 (Google AdSense) 연동 및 ads.txt 설정 FAQ

본 문서는 CreAibox 멀티테넌시(서브도메인 및 커스텀 독립 도메인) 환경에서 구글 애드센스를 설정하고 `ads.txt` 파일을 서비스하는 방법에 대한 자주 묻는 질문과 가이드를 다룹니다.

---

## Q1. 구글 애드센스 광고를 내 블로그에 노출하려면 어떻게 하나요?

1. **애드센스 가입 및 사이트 등록**:
   - 구글 애드센스 대시보드([adsense.google.com](https://adsense.google.com))에 접속하여 본인의 도메인(예: `golfgosu.net`, `guidenara.com`)을 사이트로 추가합니다.
2. **게시자 ID 복사**:
   - 애드센스 대시보드 화면 상단이나 코드 상에 노출된 본인의 고유 게시자 ID(예: `pub-3034792021993631`)를 복사합니다.
3. **블로그 관리 콘솔 입력**:
   - CreAibox 스튜디오의 블로그 관리 화면 ➡️ **[SEO 및 애널리틱스 연동]** 탭으로 이동합니다.
   - 복사한 `pub-`로 시작하는 16자리 애드센스 게시자 ID를 입력하고 **저장**합니다.
4. **광고 코드 삽입**:
   - 애드센스 광고 단위 코드를 본문이나 위젯에 연동하여 광고를 활성화합니다.

---

## Q2. ads.txt 파일이란 무엇이고, 왜 등록해야 하나요?

`ads.txt`는 해당 도메인에서 광고를 송출할 권한이 있는 판매자(게시자)임을 구글 애드센스에 알리는 텍스트 파일입니다.
이 파일이 사이트의 루트 경로(예: `https://golfgosu.net/ads.txt`)에서 제대로 서비스되지 않으면 애드센스에서 "수익 손실 위험 - ads.txt 문제를 해결해야 합니다."라는 경고가 발생하며 광고 게재가 차단될 수 있습니다.

---

## Q3. CreAibox에서 ads.txt 파일은 어떻게 서비스되나요? 따로 업로드해야 하나요?

**아닙니다. 파일 업로드가 전혀 필요하지 않으며 완전히 자동화되어 있습니다.**

CreAibox는 Next.js 미들웨어(Middleware)와 동적 라우팅 기술을 사용하여 작동합니다:
1. 구글 크롤러나 방문자가 `https://yourdomain.com/ads.txt`로 접근합니다.
2. Next.js 미들웨어(`middleware.ts`)가 이 요청을 가로채어 내부 정적 검사에서 제외하고 동적으로 관리되는 브랜드 API 경로(`/brand/[brand_id]/ads.txt`)로 토스합니다.
3. API 서버는 데이터베이스(`profiles` 테이블의 `extra_configs`)를 실시간 조회하여 해당 블로그 주인이 입력해 둔 애드센스 게시자 ID를 가져옵니다.
4. 최종적으로 구글 공식 규격에 맞는 표준 텍스트를 `text/plain` 포맷으로 즉시 브라우저와 구글 봇에게 반환합니다:
   ```text
   google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0
   ```

---

## Q4. 게시자 ID를 입력했는데도 애드센스 대시보드에 승인 대기나 오류로 떠요.

* **반영 시간 지연**: 구글 애드센스 크롤러가 `ads.txt` 파일을 다시 긁어가는 데는 보통 수 시간에서 길게는 며칠(최대 3~7일)까지 소요될 수 있습니다. 애드센스 사이트 관리 화면에서 **[업데이트 확인]** 단추를 눌러 수동 크롤링을 유도하실 수 있습니다.
* **ID 입력 오류**: 게시자 ID 입력 시 앞에 `pub-` 글자가 누락되었거나 숫자가 오타 없이 정확하게 입력되었는지 확인해 주세요.
