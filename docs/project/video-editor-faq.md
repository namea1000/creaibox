# 비디오 스튜디오(Video Studio) 오디오 에셋 및 내보내기 FAQ

본 문서는 비디오 스튜디오에서 영상 편집 시 사용되는 오디오 포맷(WAV, MP3)의 차이점과 인-브라우저 내보내기(Export) 환경에서의 스토리지 및 메모리 제약에 대한 자주 묻는 질문(FAQ)을 다룹니다.

---

## Q1. 비디오 스튜디오에서 영상 제작 시 WAV 고음질 파일과 MP3 파일 중 무엇을 사용하는 것이 좋나요?

**MP3 파일 사용을 강력히 권장합니다.**

로컬 환경에서 직접 편집을 진행하더라도, 용량이 1/10 수준으로 압축된 고품질 MP3(예: 320kbps)를 소스로 사용하는 것이 시스템 안정성과 쾌적한 편집 속도를 확보하는 데 압도적으로 유리합니다.

---

## Q2. 고음질 WAV 파일로 비디오를 제작해 내보내기(Export)하면 최종 비디오(.mp4) 용량도 더 커지나요?

**아닙니다. 최종 영상의 파일 용량은 소스 포맷(WAV 또는 MP3)과 무관하게 완전히 동일합니다.**

비디오 스튜디오에서 최종 비디오를 `내보내기(Export)` 할 때, 타임라인의 오디오 트랙은 원본 포맷에 상관없이 **에디터가 지정한 규격(예: AAC 코덱, 160kbps 비트레이트)**으로 새롭게 인코딩(재압축)되어 영상 파일 내부에 병합됩니다.
즉, 초당 저장되는 오디오 데이터의 양이 고정되므로 원본이 100MB짜리 WAV이든, 10MB짜리 MP3이든 간에 완성된 영상 파일의 오디오 영역 크기는 동일합니다.

---

## Q3. WAV 파일과 MP3 파일이 브라우저에서 실행될 때 메모리(RAM) 점유율 차이가 있나요?

**아닙니다. 브라우저가 오디오를 디코딩하고 메모리에 올리는 순간, 두 포맷의 메모리 사용량은 완전히 같아집니다.**

브라우저의 Web Audio API는 음악을 연주하거나 처리하기 위해 파일의 압축을 완전히 해제하여 무압축 상태의 실수 배열(Float32 `AudioBuffer`)로 메모리에 올립니다. 
따라서 파일 형태일 때는 MP3가 5MB, WAV가 50MB로 차이가 크더라도, **디코딩된 후 메모리에 탑재되었을 때는 재생 시간과 주파수가 같다면 완벽히 동일한 메모리 용량을 차지**하게 됩니다.

---

## Q4. 메모리 사용량이 같다면, 왜 로컬 편집 상황에서도 MP3 사용을 권장하는 건가요?

웹 브라우저의 저장 메커니즘과 로컬 파일 관리 효율 때문입니다:

1. **브라우저 로컬 저장소(IndexedDB)의 용량 제한(Quota)**:
   - 사용자가 올린 오디오/비디오 원본 파일들은 새로고침을 하거나 나중에 다시 들어와도 유지될 수 있도록 브라우저 데이터베이스인 **IndexedDB**에 백업 보관됩니다.
   - 브라우저는 단일 사이트가 하드디스크의 일정량 이상을 쓰지 못하도록 보안 한계를 걸고 있기 때문에, 기가바이트 단위의 거대한 WAV 파일들을 여러 개 올리면 **저장 공간 한계 초과 에러(QuotaExceededError)**가 발생해 편집 진행 상황이 저장되지 않을 수 있습니다.
2. **최초 드래그 업로드 및 로딩 속도**:
   - 편집기에 파일을 처음 등록할 때 파일을 읽고 IndexedDB에 쓰게 되는데, 500MB짜리 WAV보다 50MB짜리 MP3를 처리할 때 **읽기/쓰기 디스크 속도가 10배 이상 빠르고 쾌적**하며 브라우저 랙 현상이 전혀 없습니다.
3. **프로젝트 백업 편의성**:
   - 편집 내역을 JSON 파일 등으로 백업하거나 다른 기기로 이동할 때, 소스 에셋이 무거운 WAV라면 백업 파일 자체가 지나치게 커져 복원과 관리가 곤란해집니다.

---

## Q5. 사용자 PC로 바로 저장하게 되어 있는데도 브라우저 저장 한계 초과를 걱정해야 하나요?

**최종 내보내기 결과물**은 사용자 디스크로 직접 스트리밍 쓰기(FileSystem API)를 하기 때문에 크기 제한 없이 안전하게 내보낼 수 있습니다. 
하지만 **편집 도중(진행 상황 자동 저장)**에는 브라우저 내부 IndexedDB 데이터베이스에 원본 미디어가 백업 보관되므로, 원본 음원이 WAV처럼 과도하게 무거우면 **편집 중에 데이터베이스 저장 한계**에 부딪힐 수 있습니다.

따라서 쾌적하고 안전한 영상 편집을 위해서는 **음악 파일은 무조건 MP3 포맷으로 변환하여 작업하시는 것이 가장 바람직한 결론**입니다.
