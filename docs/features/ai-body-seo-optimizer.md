# AI 본문 SEO 자동 교정 엔진 기술 명세서

본 문서는 크리에이박스(CreAiBox) 스튜디오 에디터 내에 구현된 **AI 본문 SEO 자동 교정 엔진**의 작동 원리, 아키텍처 설계 사상, 프롬프트 규칙 가이드라인 및 개발자 참고 사항을 기술합니다.

---

## 1. 아키텍처 개요 (Architecture Overview)

작성자가 공들여 쓴 포스트 본문의 원래 문맥과 핵심 정보(어조, 뉘앙스)를 훼손하지 않으면서, Rank Math SEO 분석 기준에 맞는 소제목 구조화 및 키워드 최적화 배치를 자동화하는 지능형 서브 엔진입니다.

외부 노출 스니펫(Title, Meta Description, Slug)을 다듬어 주는 상단 보라색 `[AI SEO 최적화 자동 생성]` 버튼과 역할을 철저히 분리하여, **"원고 본문 내 텍스트 가공"**만을 안전하게 단독으로 전담합니다.

```mermaid
graph TD

    C --> D[Gemini 3.1 Flash Lite API 호출]

    F --> G[기존 주석 스키마 & 에디토리얼 설정 추출 백업]
    G --> H[교정 본문 + 백업 주석 재합병 (Merge)]
    H --> I[에디터 본문 데이터 교체 및 렌더링 완료]
```

## 2. 4대 핵심 교정 미션 (Core Optimization Rules)

AI가 본문 텍스트를 정돈할 때 지켜야 하는 핵심 SEO 규격은 다음과 같습니다:

1. **포커스 키워드 초기 배치 (Early Keyword Placement)**:
   * 본문 전체 텍스트의 시작 구간(첫 10% 이내 또는 첫 문단 내)에 포커스 키워드를 자연스럽게 흐름에 맞추어 1회 삽입하도록 튜닝합니다.
2. **부제목 키워드 배치 (Keyword in Headings)**:
   * `<h2>` 또는 `<h3>` 소제목 중 최소 1~2개 이상에 포커스 키워드를 문맥에 어울리는 형태로 주입합니다. (가독성이 해쳐지지 않도록 적절히 조사를 결합하거나 타이틀을 자연스럽게 튜닝하는 방식을 허용합니다.)
3. **키워드 밀도 조율 (Keyword Density Balancing)**:
   * 본문 내의 포커스 키워드 빈도가 **1.0% 전후 (0.8% ~ 1.5% 수준)**가 되도록 모니터링 조율합니다.
   * 키워드가 과도하게 밀집된 문단은 동의어로 완화시키고, 너무 부족한 경우 문맥의 흐름에 맞춰 1~2회 보충하여 점수를 확보합니다.
4. **소제목 논리 구조화 (Logical Heading Hierarchy)**:
   * `<h2>`, `<h3>` 등의 소제목 위계질서가 한 권의 책 목차처럼 구조적으로 일관되게 정돈되도록 지원합니다.

## 3. 스키마 및 에디토리얼 데이터 보존 가드 (Comment Bypass Guard)

가장 중요한 기술적 안전장치로, AI가 교정한 새 본문 HTML로 에디터 상태를 교체할 때 **기존 본문에 이미 장착되어 있던 다른 중요 메타데이터(스키마 주석 등)가 유실되는 순환 오류를 차단**합니다.

### 작동 메커니즘
1. AI에 본문을 보내기 전 혹은 완료 직후, 기존 본문에서 아래의 두 정규식을 사용해 주석 블록을 싹 긁어와 메모리에 일시 백업합니다.
   * **구조화 스키마 주석**: `/<!--\s*CREAIBOX_SCHEMA_START([\s\S]*?)CREAIBOX_SCHEMA_END\s*-->/gi`
   * **에디토리얼 설정 주석**: `/<!--\s*CREAIBOX_EDITORIAL_START([\s\S]*?)CREAIBOX_EDITORIAL_END\s*-->/gi`
2. AI가 전달해 준 원시 교정 본문 HTML에서 불필요하게 중복 유입된 주석 태그들을 한 번 더 정화(Clean)해 줍니다.
3. 교정 본문의 가장 하단에 백업해 두었던 주석 블록들을 차례대로 **재합병(Merge)**하여 `updateLocalData` 상태 업데이트를 최종 갱신합니다.
4. 이를 통해 사용자가 꽂아 두었던 `FAQPage`나 `Article` 등의 스키마 마크업 배지가 글자 교정 후에도 화면에 안정적으로 100% 보존됩니다.
